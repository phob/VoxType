# Product

## Register

product

## Users

VoxType is for Windows users who want fast, reliable dictation across real desktop apps without sending core speech, screen, dictionary, or insertion data to the cloud. The primary experience should be beginner-friendly first: new users should be able to install VoxType, choose a model, set a hotkey, and dictate into another app without understanding transcription engines, OCR diagnostics, native helper details, or insertion internals.

Important user groups include Windows power users, privacy-conscious users, developers and technical users with specialized vocabulary, and support, sales, operations, or admin workers who type repetitive text into office apps, CRMs, ticket systems, remote sessions, virtual machines, and customer environments.

## Product Purpose

VoxType turns speech into accurate text anywhere on Windows using local models, screen-aware context, app profiles, personal vocabulary, and reliable insertion modes. It should feel like the dependable Windows dictation tool users can leave running in the background: press a global hotkey, speak naturally, and have text land in the active app with the right local context.

Success means the app handles setup clearly, protects privacy by default, explains blockers in plain language, and hides dense diagnostics until they are actually needed. The product is not a generic Whisper UI; it is a local Windows dictation system that understands the target app, visible screen text, insertion constraints, and user vocabulary.

## Brand Personality

Calm, secure, capable.

The interface should feel steady and trustworthy rather than flashy. It can be technical when needed, but the default release surface should remain composed, practical, and approachable for people who primarily want dictation to work. VoxType should communicate quiet confidence: local-first privacy, robust Windows behavior, and smart setup defaults without theatrical AI branding.

## Anti-references

VoxType should not look or feel like a macOS clone. Avoid translucent macOS-style chrome, traffic-light-window metaphors, floating frosted panels, generic Apple-like settings surfaces, or visual language that hides the fact that this is a Windows-first tool.

Also avoid generic AI SaaS styling, gamer utility aesthetics, neon cyberpunk dashboards, and sterile enterprise admin pages. The UI should not expose implementation complexity as product personality.

## Design Principles

1. Beginner-friendly first, expert-capable underneath.
2. Make privacy and local control visible without making the user manage internals.
3. Design for real Windows work, including messy third-party apps, remote sessions, and legacy controls.
4. Keep the everyday surface focused on setup readiness and confidence, not in-app recording controls.
5. Explain blockers and fallback behavior in plain language before exposing diagnostics.

## Accessibility & Inclusion

Design for clear text, strong contrast, keyboard-accessible controls, and predictable focus behavior. Motion should be subtle and nonessential, with no workflow depending on animation. Because VoxType is used while users move between other Windows apps, status, error, and setup states should be scannable, concise, and understandable without sustained attention.
