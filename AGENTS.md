# VoxType

The Goal of VoxType is to be a state of the art Voice recognition software for Windows. It will use the most modern offline voice recognition models widely available to the User.

## Pull Requests

When creating or drafting a GitHub pull request for this repository, use `.github/PULL_REQUEST_TEMPLATE.md` as the PR body structure.

- Fill `## Summary` with concise implementation-level bullets.
- Fill `## Release Notes` with user-facing changelog bullets that are suitable for the public GitHub Release notes.
- Use optional `### Added`, `### Fixed`, `### Improved`, `### Changed`, `### Documentation`, or `### Internal Changes` subsections inside `## Release Notes` when a PR needs multiple public changelog bullets.
- Fill `## Checks` with the actual validation performed, marking checkboxes only when that check passed.
- Choose a PR title that reads well in generated release notes.
- If the change should not appear in public release notes, say that in `## Release Notes` and apply or request the `skip-changelog` label.
